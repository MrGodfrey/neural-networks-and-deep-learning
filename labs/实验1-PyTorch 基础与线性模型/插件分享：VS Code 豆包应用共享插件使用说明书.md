# VS Code 豆包应用共享插件使用说明书

本文内容来自 [@Xxxx37](https://github.com/Xxxx37) 的分享，感谢他的贡献。

## 一、插件概述

豆包应用共享插件（Doubao App Share VS Code Plugin）是字节跳动为 VS Code 开发的官方扩展，需要配合豆包电脑版客户端使用，不能只使用网页版豆包。

它的核心功能是将 VS Code 中当前打开的代码文件上下文自动共享给豆包。使用后，不需要手动复制、粘贴代码，豆包可以基于当前文件内容进行答疑、分析和代码生成。

核心价值：

- 免复制粘贴：自动同步 VS Code 当前激活文件内容到豆包。
- 精准上下文：以当前激活 Tab 为对话上下文，切换文件后自动更新。
- 高效协作：支持代码提问、问题排查和代码生成。

优点：当 VS Code 自带 Chat 额度受限时，可以不用再把整个文件复制到其他大模型工具中。

缺点：插件体积偏大，部分文件可能无法识别，也不能直接替你修改代码。

## 二、安装步骤

1. 打开 VS Code，点击左侧活动栏中的“扩展”图标，或使用快捷键 `Ctrl+Shift+X`。
2. 在搜索框中输入“豆包应用共享插件”。
3. 找到官方插件（开发者为 Doubao），点击“安装”。
4. 安装完成后，重启 VS Code 使插件生效。

## 三、使用流程

### 前提条件

- 已安装豆包电脑版客户端，而不是网页版。
- 已在 VS Code 中安装“豆包应用共享插件”。
- 已登录豆包客户端账号。

### 步骤 1：启动共享

1. 打开 VS Code，确保待分析的代码文件是当前激活 Tab。
2. 打开豆包电脑版，点击浮窗或聊天框左下角的“+”，选择“共享应用或屏幕”。
3. 在应用列表中选择 Visual Studio Code，并确认共享。
4. 共享成功后，豆包会自动获取当前文件内容。

### 步骤 2：对话与操作

基础提问示例：

- “解释这段代码的功能。”
- “找出代码中的 bug。”
- “优化这段代码的性能。”
- “给代码添加详细注释。”

代码生成示例：

- “补全这个函数的逻辑。”
- “为当前文件写单元测试。”

上下文切换：

- 切换 VS Code 当前激活 Tab 后，豆包会自动同步新文件内容。
- 关闭当前文件后，共享上下文会自动清空。

### 步骤 3：停止共享

- 方式一：关闭 VS Code 或豆包客户端，自动终止共享。
- 方式二：在豆包浮窗中进入“共享应用或屏幕”，选择停止共享。

## 四、功能详解

共享范围：

- 支持共享当前激活的单个文件完整内容。
- 支持常见编程语言和文本文件，例如 Python、Java、C++、LaTeX 等。
- 文件名、路径和代码结构会随文件内容一起同步。
- 不支持共享多个文件、整个项目、终端或控制台内容。

核心能力：

- 代码分析：解释代码逻辑、复杂度和依赖关系。
- 问题排查：定位语法错误、逻辑漏洞和异常风险。
- 代码生成：补全、重构、优化代码或添加注释。

常见状态：

- “已共享”：连接正常，可以正常提问。
- “未找到 VS Code”：插件未安装，或 VS Code 未启动。
- “无激活文件”：VS Code 当前没有打开可共享的文件。

## 五、常见问题与解决

**Q1：共享时提示“未找到 VS Code”。**

确认 VS Code 已安装插件并重启。必要时关闭 VS Code 后重新打开，或重新安装插件。

**Q2：豆包获取不到文件内容。**

确认目标文件是 VS Code 的当前激活 Tab。可以关闭豆包后重新开启共享，并检查插件是否为最新版。

**Q3：共享后内容不同步。**

手动切换一次 VS Code Tab 触发同步；如果仍不同步，关闭共享后重新开启。

**Q4：Dev Container 环境无法使用。**

将插件更新到 `0.1.1` 或更高版本。

## 六、最佳实践

- 提问时尽量使用“明确指令 + 当前代码上下文”的方式，例如：“基于当前文件，优化 XX 函数并添加注释。”
- 不使用时及时停止共享，减少不必要的上下文暴露。
- 保持 VS Code、豆包客户端和插件为最新版。
- 涉及敏感代码时，建议关闭共享或先确认共享范围。

## 七、卸载插件

1. 在 VS Code 中打开“扩展”。
2. 找到“豆包应用共享插件”。
3. 点击“卸载”。
4. 重启 VS Code。
