# 深度学习实验环境配置指南（多平台支持）

本指南适用于 macOS（支持 Apple Silicon M 系列芯片及 Intel 芯片）、Windows 和 Linux 操作系统。我们将以 Miniconda 管理 Python 环境，并安装 PyTorch。由于这门课程涉及大量的代码操作，我们将借助“终端”（Terminal）和 VS Code 作为主要工具。

---

## 零、基础概念预警（小白必看）

在开始之前，为了让你能清晰地知道每一步在做什么，请先了解以下概念：

1. **什么是“终端（Terminal）”？为什么要用它？**
   * **概念**：终端是你直接与计算机“对话”的窗口。平时你通过点击鼠标、双击图标来告诉电脑打开软件，而在终端里，你通过输入代码（命令行）来下达指令。对于 Windows 系统，你可以使用 `PowerShell` 或 `命令提示符 (cmd)`，甚至安装 `Git Bash` 或 `WSL (Windows Subsystem for Linux)`。对于 Mac/Linux，请在应用中搜索“终端”。
   * **原因**：在编程和深度学习领域，很多工具（如包管理器、Python 解释器等）没有图形化界面。使用终端可以直接、高效地下载代码、安装环境、启动服务，是每个开发者必备的基本功。

2. **什么是“虚拟环境（Virtual Environment）”？**
   * **概念**：假设你有两个不同的深度学习项目，项目 A 需要 Python 3.8，项目 B 需要 Python 3.10。如果都在电脑全局安装，会产生冲突。虚拟环境就像是为每个项目创建的“独立房间”，在房间 A 里装的材料，不会影响房间 B。
   * **在 Conda 中的体现**：Conda 是一个环境管理器。使用 `conda create` 命令，就能轻松造出一个针对当前课程（这里我们命名为 `nndl`）的“独立房间”。

3. **什么是 GPU 加速 (CUDA / MPS)？**
   * **概念**：深度学习涉及大量的矩阵运算，只用 CPU 算会非常慢，所以我们通常借助显卡（GPU）来计算。
   * **CUDA**：大部分配置了 NVIDIA 独立显卡的 Windows 和 Linux 电脑，使用的是 NVIDIA 开发的 CUDA 架构加速。
   * **MPS**：Metal Performance Shaders (MPS) 是苹果提供的接口，它能让 PyTorch 调用 Apple 芯片（M1/M2/M3 等）的 GPU 进行加速计算。

## 一、安装 Miniconda

1. 前往 [Miniconda 官方下载页](https://docs.conda.io/en/latest/miniconda.html)，选择对应系统：
   - **Mac (Apple Silicon)**：选 `Miniconda3 macOS Apple M1 64-bit pkg` 或 `.sh`
   - **Mac (Intel)**：选 `Miniconda3 macOS Intel x86 64-bit pkg` 或 `.sh`
   - **Windows**：选 `Miniconda3 Windows 64-bit` 安装程序 (.exe)，双击运行。重要提示：请勾选“Add Miniconda to my PATH environment variable”或安装后手动在搜索栏打开 "Anaconda Prompt"。
   - **Linux**：下载 `.sh` 脚本并运行 `bash Miniconda3-latest-Linux-x86_64.sh`。

2. Mac/Linux 运行脚本安装（如使用 `.sh`）：

   ```bash
   bash ~/Downloads/Miniconda3-latest-MacOSX-arm64.sh   # 替换为你实际的脚本名
   ```

3. 完成安装后，Windows 用户打开“Anaconda Prompt”或在 PowerShell 中输入，Mac/Linux 用户直接在终端中输入：

   ```bash
   conda --version
   ```
   如果输出版本号，说明成功。

## 二、创建虚拟环境

我们专门为这门课程创建一个名为 `nndl` 的虚拟环境。建议使用 Python 3.10，与 PyTorch 及各依赖包兼容性最好：

```bash
# create -n nndl 代表创建一个名为 nndl 的环境，指定 python 版本为 3.10
conda create -n nndl python=3.10 -y

# 激活我们要用的环境
conda activate nndl
```
*(注意：每次做实验前，如果重新打开了终端（或 Anaconda Prompt），都需要执行 `conda activate nndl` 激活环境！)*

## 三、安装 PyTorch

请根据你的操作系统和硬件选择正确的安装命令：

### Windows / Linux (带 Nvidia 显卡)

你需要安装支持 CUDA 的 PyTorch（建议根据你的显卡驱动，选择对应的 CUDA 版本，如 11.8 或 12.1）。在官网获取最新命令，一般如下：
```bash
# 以 CUDA 12.1 为例：
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

验证 CUDA 加速是否可用：
```python
import torch
print(torch.cuda.is_available())  # 如果输出 True，说明显卡加速配置成功！
```

### Apple Silicon Mac (M1/M2/M3/M4)

PyTorch 原生支持 Apple Silicon 的 MPS 加速特性：
```bash
pip install torch torchvision torchaudio
```

验证 MPS 加速是否可用：
```python
import torch
print(torch.backends.mps.is_available())  # 如果输出 True 说明成功！
```

### Windows / Linux / Intel Mac (仅 CPU)

如果你电脑没有独立显卡或不使用加速，直接安装：
```bash
pip install torch torchvision torchaudio
```

## 五、将环境注册到 Jupyter

为了让 VS Code 的 Jupyter Notebook 能够识别到我们刚创建的 `nndl` 环境里的 Python 工具包，我们需要注册它：

## 六、在 VS Code 中使用 Jupyter Notebook

我们在课程中不再使用传统的浏览器版本 Jupyter，而是直接使用强大的 VS Code。

1. **安装 VS Code 插件**：
   在 VS Code 左侧点击“扩展”图标（相当于应用商店），搜索并安装以下两个官方插件：
   * **Python** (Microsoft 提供)
   * **Jupyter** (Microsoft 提供)

2. **用 VS Code 打开项目**：
   在终端中输入：
   ```bash
   cd /path/to/nndlClass   # 替换为你实际代码存放的路径
   code .                  # 用 VS Code 打开当前目录（注意 code 后有个空格加一点）
   ```
   *(如果 `code .` 命令无效，你也可以直接打开 VS Code，通过菜单 `File -> Open Folder` 手动选择实验所在的文件夹。)*

3. **打开并运行实验**：
   * 在 VS Code 左侧的资源管理器中，点击打开一个以 `.ipynb` 结尾的实验文件。
   * 文件的右上方（或代码单元格的右上方）会有一个 **“选择内核 (Select Kernel)”** 的选项。请点击它。
   * 选择 **Jupyter Kernel** -> **Python (nndl)**。
   * 现在你可以点击单元格左侧的播放键“运行”代码了！

## 八、常见问题

**Q: 运行代码时提示 `ModuleNotFoundError: No module named 'h5py'`？**  
A: 这多半是因为你没有激活环境或者没选对内核。
1. 检查 VS Code 右上角的 Jupyter 内核，是否选的是 **Python (nndl)**。
2. 如果内核是对的，说明你当时装错地方了。在终端里先执行 `conda activate nndl`，然后再运行一次 `pip install h5py`。

**Q: VS Code 选内核时找不到 `Python (nndl)`？**  
A: 先重新执行一下“第五步”里的注册命令。如果还不行，重启一下 VS Code 即可。

**Q: Apple Silicon 上 PyTorch 运行时出现奇怪的报错？**  
A: 确保在实验代码中，若要指派到 GPU 运行，使用的是 `device = torch.device("mps")`，如果写成了 `"cuda"`，在 Mac 上是跑不通的。
