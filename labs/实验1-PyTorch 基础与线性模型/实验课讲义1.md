# 实验 1 讲义总入口

这份页面是实验 1 的总导航页。你可以把它理解成一个小型 wiki 入口：不负责展开讲所有细节，而是帮你快速找到当前最需要看的那份讲义。

如果你是第一次进入这门课，建议按下面的顺序阅读：

1. 先看 [环境配置指南](环境配置指南.md)：把 Python、终端、环境和 VS Code 先接通。
2. 再看 [VS Code 与智能编程](VSCode与智能编程.md)：学会怎样在编辑器里读代码、跑代码、和 AI 协作。
3. 最后看 [Git 使用指南](Git使用指南.md)：学会保存实验历史、开分支试错，以及同步到 GitHub。

如果你已经开始做实验，也可以按自己当前遇到的问题直接跳转。

---

## 一、核心讲义入口

### 1. [环境配置指南](环境配置指南.md)

这份讲义解决的是“实验还没开始，电脑环境先卡住了”这一类问题。

它主要讲这些内容：

- 为什么环境配置要从终端讲起。
- 终端、Shell、路径、命令这些最基础的概念是什么意思。
- 如何配置 Python 环境、安装依赖，并让 VS Code 连到正确环境。

适合什么时候看：

- 第一次上这门实验课。
- 电脑还没有把 Python 环境配好。
- Notebook 跑不起来，或者 VS Code 连错了解释器 / Kernel。

读完以后，你应该能完成最关键的一步：  
**把实验真正跑起来。**

### 2. [VS Code 与智能编程](VSCode与智能编程.md)

这份讲义解决的是“环境已经配好了，但我还不会高效地写代码、看代码和用 AI 帮忙”这一类问题。

它主要讲这些内容：

- 现代编辑器到底替你解决了什么问题。
- VS Code / Trae 里最核心的工作方式是什么。
- Chat 式协作和 Agent 式协作分别适合什么任务。
- 做实验时，怎样把编辑器、终端和 AI 变成一个稳定工作流。

适合什么时候看：

- 你已经能打开项目，但还不会高效阅读和修改代码。
- 你想知道 AI 在实验课里应该怎么用才更稳。
- 你经常会问“这个报错到底该怎么读”“这个函数在哪定义”。

读完以后，你应该能完成这一步：  
**在同一个工作台里看项目、跑代码、定位问题，并更稳地借助 AI。**

### 3. [Git 使用指南](Git使用指南.md)

这份讲义解决的是“我会改代码了，但还不会保存历史、回退版本、开分支试错和同步远程仓库”这一类问题。

它主要讲这些内容：

- Git 为什么可以看成项目的时间机器。
- 快照、提交、暂存区、分支、远程仓库分别是什么意思。
- 做实验时，怎样用 Git 记录每一个清楚的小步骤。
- 当代码改崩时，怎样通过历史记录把问题范围缩小。

适合什么时候看：

- 你已经开始写实验代码，想让自己的过程更稳。
- 你担心改坏后回不去。
- 你想把代码同步到 GitHub，或者在不同电脑之间来回切换。

读完以后，你应该能完成这一步：  
**把实验过程变成一条清楚、可回退、可同步的历史。**

### 4. [有用的课程参考](有用的课程参考.md)

这份页面不是本课程的主线讲义，而是一个扩展阅读入口。

它主要收集：

- 更系统的课程资源。
- PyTorch 实战参考材料。
- AI 工具与 Agent 使用的外部学习入口。

适合什么时候看：

- 你已经完成本讲义里的主线内容，想继续往外扩展。
- 你想补更多终端、Git、开发工具或 AI 协作相关材料。
- 你做实验之外，还想进一步练习工程能力。

它的作用更像“延伸阅读区”，不是实验 1 的必经主线。

## 三、这四份讲义之间是什么关系

这四份内容不是彼此独立的，它们更像实验课里的四个层次：

1. 环境配置指南：解决“能不能开始”的问题。
2. VS Code 与智能编程：解决“怎么更高效地做”的问题。
3. Git 使用指南：解决“怎么把过程保存好、管理好”的问题。
4. 有用的课程参考：解决“做完主线后还能往哪里继续学”的问题。

如果把实验课理解成一条完整工作流，那么它大致是这样：

```text
先把环境接通 -> 再用编辑器和 AI 做实验 -> 再用 Git 管理整个过程 -> 最后用外部资料继续扩展
```

这样读下来，整套讲义会更像一个完整工具链，而不是几份互相分散的说明文档。
